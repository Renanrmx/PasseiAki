# Extension architecture

Passei Aki is a browser extension that records accesses locally and marks links that have already been visited or are partially similar.

The main code is in `src/`. The `dist/` directory is generated by `scripts/prepare-dist.mjs` and should not be edited directly.

## Main structure

- `src/worker/`: service worker/background. Centralizes persistence, matching, history, backup, import/export, downloads, support, and message handlers.
- `src/content/highlighter.js`: content script injected into pages. Collects links, queries the background, and applies visited or partial classes/styles.
- `src/panel/`: extension pages, such as the main popup, settings, history, mirrors, and exceptions.
- `src/shared/`: global helpers shared between contexts.
- `src/_locales/*/messages.json`: localized text.
- `test/`: Node tests using `node:test`.

## Script model

The project uses global extension scripts, not ES modules. This means that:

- load order matters;
- global functions are shared in the same context;
- tests load scripts with `vm` to simulate this behavior;
- before moving functions between files, confirm that all pages/manifests load the new dependency.

## Main flow of a visit

1. The background receives navigation/tab events.
2. The URL is normalized and transformed into a fingerprint.
3. The record is saved or merged via `upsertVisit`.
4. The action/icon state is updated.
5. The content script can query links on the page to know which ones should be marked.

Relevant files:

- `src/worker/background.js`
- `src/worker/background.match.js`
- `src/worker/background.database.js`
- `src/content/highlighter.js`

## Internal messages

Message types are in `src/shared/messages.js`. Whenever a new message is created:

- add the type to `AKI_MESSAGE_TYPES`;
- allow the type in the correct set in `background.js`;
- validate whether the message should be accessible by extension page, content script, or both;
- return `{ ok: true }` or `{ ok: false, error }` when the caller expects success/failure state.

## UI

The panel pages use simple HTML/CSS/JS. The `src/panel/i18n.js` helper applies `data-i18n` and `data-i18n-attr`.

When adding visible text:

- create a key in all locales;
- avoid hardcoded strings only in JS/HTML, except as a visual fallback;
- check that long text does not expand the popup width.
